Architecture overview — how the agent reads an issue and produces a pull request

End-to-End Pipeline
Trigger
Issue labeled
GitHub fires webhook when agent-fix label is applied to an issue
Step 1
Fetch issue
GitHub API: read issue title, body, and repo name
Step 2
Clone repo
git clone --depth=1 into temp dir; create feature branch
Step 3
Find files
Keyword extraction → grep → ranked candidate file list (top 20)
Step 4
ReAct loop
LLM calls tools iteratively to explore, edit, and test the fix
Step 5
Open PR
Commit changes, push branch, create ready or draft PR via GitHub API

Modules
agent/main.py
Orchestrator
Entry point. Reads env vars, wires all components together, decides ready vs. draft PR based on agent outcome and whether files were changed.
entry point pipeline control
agent/loop.py
ReAct Loop
Pure function — takes a model name, initial context, and callbacks. No GitHub or git knowledge. Calls litellm, dispatches tools, returns a LoopResult.
pure litellm max 15 iterations
agent/tools.py
Tool Implementations
Sandboxed file + shell tools. All paths validated with Path.relative_to() to prevent traversal. Shell commands allowlisted to test runners only.
sandboxed allowlisted shell no network
agent/search.py
Keyword / Grep Search
Strips stop words from the issue text, greps the repo for each keyword, deduplicates and ranks by hit count. Returns top-N file candidates for the LLM warm start.
no embeddings grep -rl
agent/github_client.py
GitHub API Client
Owns all GitHub API calls via PyGithub. Fetches issues, creates PRs (ready or draft), posts comments. No other module touches the GitHub API.
PyGithub single responsibility
agent/repo.py
Git Repo Manager
Owns all git operations: shallow clone, branch creation, commit, and push. Uses token-authenticated remote URL. Cleans up temp directory on exit.
subprocess git auto cleanup
PR Outcome Decision
Agent says Files changed? Outcome
finish(done) Yes ✅ Ready PR — "Closes #N", label: agent-generated
finish(done) No ⚠️ Draft PR — treated as uncertain (no change = no fix)
finish(uncertain) Yes 📝 Draft PR + comment on issue: "Agent opened draft PR #N"
finish(uncertain) No 💬 Comment on issue only: "Agent could not fix this issue"
Max iterations hit Any Same as finish(uncertain)

Environment Variables
GITHUB_TOKEN required
→ secrets.GITHUB_TOKEN
GitHub Actions token with contents:write, pull-requests:write, issues:write
REPO_NAME required
→ github.repository
Full repo name, e.g. owner/repo
ISSUE_NUMBER required
→ github.event.issue.number
Issue number that triggered the workflow
LLM_PROVIDER required
anthropic/claude-sonnet-4-6
Any litellm-compatible model string. Controls which LLM the agent uses.
ANTHROPIC_API_KEY if Anthropic
sk-ant-…
Required when LLM_PROVIDER is an Anthropic model
OPENAI_API_KEY if OpenAI
sk-…
Required when LLM_PROVIDER is an OpenAI model
MAX_ITERATIONS optional
15 (default)
Hard cap on ReAct loop iterations before treating as uncertain
